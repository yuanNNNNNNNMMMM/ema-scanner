# 📌 PART 3: 自動推播設定

## 步驟1: 啟用Cron Job

### 1.1 進入Service設定
1. Zeabur Dashboard → **ema-scanner** Service
2. 點 **"設定"** 標籤
3. 捲下來找 **"定時自動運載"** 選項

### 1.2 啟用Cron
1. 點灰色的 **圓形開關**，變成紫色表示已啟用
2. 應該會出現時間設定選項

### 1.3 設定執行時間

**Zeabur只支持一個固定時間，所以選擇你想要的間隔：**

#### 選項A: 每4小時 (推薦)
```
0 */4 * * *
```
= 每4小時執行一次 (UTC時間)
= 台灣時間: 08:00, 12:00, 16:00, 20:00, 00:00, 04:00

#### 選項B: 每12小時
```
0 0,12 * * *
```
= 每天 UTC 00:00 和 12:00
= 台灣時間: 08:00 和 20:00

#### 選項C: 每天
```
0 0 * * *
```
= 每天 UTC 00:00
= 台灣時間: 08:00 (每天早上)

### 1.4 保存設定
1. 輸入你選擇的Cron表達式
2. 點 **"儲存"** 按鈕
3. Zeabur會自動應用設定

## 步驟2: 測試推播 (可選但建議)

### 2.1 手動觸發一次掃描

在Zeabur Dashboard中：

1. 進入 **ema-scanner** Service
2. 點右上角 **"..."** 菜單
3. 選 **"運行"** 或 **"Execute"**
4. 系統會立即執行一次掃描

### 2.2 檢查結果

#### 檢查方式1: 查看Zeabur Logs
1. 點 **"日誌"** 標籤
2. 尋找 "✅ Telegram 消息已發送" 訊息
3. 沒有錯誤訊息 = 成功！

#### 檢查方式2: 查看Telegram
1. 打開Telegram
2. 找 **@ema_scanner_tw_bot** 聊天
3. 應該收到掃描結果

## 📅 Cron表達式說明

### 格式: `分 小時 日 月 星期`

```
0 */4 * * *
│ │   │ │ │
│ │   │ │ └─ 星期 (0-6, 0=週日)
│ │   │ └─── 月份 (1-12)
│ │   └───── 日期 (1-31)
│ └───────── 小時 (0-23)
└─────────── 分鐘 (0-59)
```

### 常用例子

| 表達式 | 說明 | 台灣時間 |
|--------|------|---------|
| `0 */4 * * *` | 每4小時 | 08:00, 12:00, 16:00, 20:00, 00:00, 04:00 |
| `0 0,12 * * *` | 每12小時 | 08:00, 20:00 |
| `0 0 * * *` | 每天 | 08:00 |
| `0 8 * * *` | 每天08:00 (UTC) | 16:00 (台灣) |
| `0 0 * * 1` | 每週一 | 08:00 (台灣) |

## 🐛 除錯

### 問題1: 沒有收到Telegram推播

**檢查清單:**

1. **Cron Job是否已啟用？**
   - 進入設定，檢查開關是否為紫色

2. **時間到了嗎？**
   - 現在時間是否符合Cron表達式?
   - UTC 00:00 = 台灣 08:00
   - 時差是8小時

3. **查看Zeabur Logs**
   ```
   選項A: 看到 "✅ Telegram 消息已發送" ✅
   選項B: 看到 "❌" 錯誤訊息
   ```

4. **確認環境變數**
   - BOT_TOKEN 正確嗎?
   - CHAT_ID 正確嗎?
   - 沒有多餘空格?

5. **Bot是否在你的聊天中？**
   - 打開Telegram
   - 搜尋 @ema_scanner_tw_bot
   - 發送 `/start` 確認Bot運作

### 問題2: Cron Job沒有執行

**解決:**

1. 檢查表達式語法是否正確
2. 確保已點 "儲存"
3. 等待下一個排程時間

### 問題3: 看到Logs錯誤 "Cannot find module"

**解決:**

1. 點 "重新Build"
2. 等待npm重新安裝依賴
3. 重新測試
